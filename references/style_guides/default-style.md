# Default Newsletter Style Guide

This style guide applies to all episodes in the newsletter series unless a custom style guide is provided by the user.

## Tone & Voice
- **Tone**: Professional yet conversational (Senior Peer Programmer / Expert Guide).
- **Voice**: Active voice, direct, and concise. Avoid jargon where simple words suffice.
- **Clarity**: Prioritize technical accuracy over marketing flair.

## Structural Requirements
- **Title**: Every episode must start with an H1 title (`# Episode XX: Title`).
- **Overview**: A brief (2-3 sentence) summary of what will be covered.
- **Main Content**: 300-500 words per episode, broken into logical H2 and H3 subheadings.
- **Visuals**: At least one Mermaid diagram or Markdown table per episode.
- **Key Takeaways**: A bulleted list of 3-4 critical points at the end.
- **Navigation**: Hyperlinks to previous and next episodes at the bottom.

## Formatting Standards
- **Callouts**: Use `> **Note**: ...` for important asides.
- **Bolding**: Use **bold** for key terms, definitions, and specific legal/technical references.
- **Code Blocks**: Use for technical commands or code examples.
- **Links**: Use descriptive link text (e.g., `[Episode 01](01-slug.md)` instead of `[Click here](...)`).

## Prohibitions
- **No Filler**: Avoid generic introductions ("In this world of technology...").
- **No AI Narratives**: Do not use "As an AI..." or "I hope this helps...".
- **No Redundancy**: If a topic is covered in detail in a previous episode, link to it instead of repeating the full explanation.

# Richard Feynman Newsletter Style Guide

This style guide is inspired by Richard Feynman's approach to learning and communication.

## Tone & Voice
- **Tone**: Energetic, inquisitive, and fundamentally simple.
- **Voice**: "Explain it to me like I'm a curious adult."
- **Analogy First**: Every complex concept MUST be introduced with a simple, everyday analogy (e.g., "Imagine a library with only one book...").
- **No Jargon**: If a technical term is necessary, define it immediately using plain English. Never hide behind big words.

## Structural Requirements
- **The Hook**: Start each episode with a fundamental question (e.g., "Why does it work this way?").
- **The Build-Up**: Layer information from the simplest part to the most complex.
- **Visuals**: Use Mermaid diagrams to show *how* things move or relate, not just what they are.
- **The "Aha!" Moment**: Every episode should lead the reader to a clear, intuitive realization.

## Formatting Standards
- **Simple Tables**: Use tables only for simple side-by-side comparisons.
- **Conversational Callouts**: Use `> **Think about it**: ...` to prompt the reader's intuition.
- **Active Dialogue**: Write as if you are having a conversation with the reader at a blackboard.

## Prohibitions
- **No Complexity for Complexity's Sake**: If you can't explain it simply, you don't understand it yet.
- **No Formalism**: Avoid overly formal, academic language.
- **No Passive Voice**: Keep it active and moving.
